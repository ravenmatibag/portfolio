# Contributing

Thanks for your interest in this project! This is a personal portfolio site, so the scope for outside contributions is intentionally small — but bug reports, accessibility fixes, and performance suggestions are welcome.

## What's welcome

- Bug fixes (broken links, layout issues, console errors)
- Accessibility improvements
- Performance improvements (image optimization, unused CSS/JS cleanup)
- Documentation fixes (typos, unclear setup steps)

## What's not in scope

- Redesigns or changes to the visual style, layout, or color palette
- Changes to personal content (bio, resume, project descriptions, contact info)

Content and design changes are reserved for the site owner.

## Getting started

1. Fork the repository.
2. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/portfolio.git
   ```
3. Open `index.html` directly in a browser, or serve the folder locally (see [README.md](README.md#installation)).
4. Make your changes on a new branch:
   ```bash
   git checkout -b fix/short-description
   ```
5. Test in a browser — check desktop and mobile widths, and confirm no console errors.
6. Commit with a clear message and open a pull request describing the change and why it's needed.

## Code style

- Match the existing formatting (2-space indentation, existing naming conventions).
- Keep vanilla HTML/CSS/JS — no build tooling or frameworks are used in this project.
- Comment non-obvious logic, consistent with the existing code.

## Reporting issues

Open a GitHub issue with:
- A clear description of the problem
- Steps to reproduce (if applicable)
- Browser/device, if it's a rendering issue
- Screenshots, if helpful
